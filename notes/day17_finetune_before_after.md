# Day17：微调前后模型对比评估

## 1. 实验目的

Day16 已经完成了第一次小规模微调：

```text
yolo11s.pt
-> MOT17 小数据集微调
-> best.pt
```

Day17 的目标是验证：

```text
微调后的 best.pt 在完整 MOT17-02-SDP 序列上，是否真的比预训练 yolo11s.pt 更适合当前场景。
```

注意：这里使用完整 600 帧 MOT17-02-SDP，并使用同一个评估脚本：

```text
scripts/evaluate_mot_detections.py
```

这样才能和之前的 baseline 公平对比。

## 2. 对比设置

微调前模型：

```text
yolo11s.pt
```

微调后模型：

```text
runs/detect/experiments/day16_train_mot17_yolo_small/yolo11s_imgsz960_epochs5/weights/best.pt
```

固定参数：

```text
imgsz = 960
conf = 0.25
nms_iou = 0.70
eval_iou = 0.50
```

评估数据：

```text
MOT17-02-SDP 完整 600 帧
GT boxes = 18581
```

## 3. 整体结果

| model | predictions | GT | TP | FP | FN | Precision | Recall | Avg IoU |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| pretrained_yolo11s_imgsz960 | 8815 | 18581 | 6771 | 2044 | 11810 | 0.7681 | 0.3644 | 0.8067 |
| finetuned_best_imgsz960 | 19375 | 18581 | 12877 | 6498 | 5704 | 0.6646 | 0.6930 | 0.7992 |

## 4. 整体结果解释

微调后：

```text
predictions: 8815 -> 19375，增加 10560
TP:          6771 -> 12877，增加 6106
FN:         11810 -> 5704，减少 6106
Recall:   0.3644 -> 0.6930，大幅提高
```

这说明：

```text
微调让模型更适应 MOT17 行人场景，找回了大量原本漏检的行人。
```

但同时：

```text
FP:         2044 -> 6498，增加 4454
Precision:0.7681 -> 0.6646，明显下降
```

这说明：

```text
微调后的模型变得更激进，输出了更多 person 框。
其中很多新框确实匹配到了 GT，但也引入了更多误检。
```

Avg IoU：

```text
0.8067 -> 0.7992
```

略微下降，说明新增 TP 中可能包含更多困难目标、小目标或遮挡目标，它们的定位质量略低。

## 7. 微调是否有效

从 Recall 角度看，微调非常有效：

```text
Recall: 0.3644 -> 0.6930
FN: 11810 -> 5704
```

这说明微调确实让模型适应了 MOT17 的行人场景。

从 Precision 角度看，微调有副作用：

```text
Precision: 0.7681 -> 0.6646
FP: 2044 -> 6498
```

这说明模型变得更容易输出 person 框，误检也增加。

所以最终判断不是“微调绝对更好”，而是：

```text
微调显著提高 Recall，但牺牲了 Precision。
```

## 8. 为什么会这样

微调数据集中只有 `person` 一个类别，并且来自 MOT17 行人场景。

训练后模型会更倾向于：

```text
把小目标、遮挡目标、密集人群中的局部区域判为 person。
```

这有利于减少漏检：

```text
FN 减少，Recall 上升。
```

但也可能导致：

```text
一些背景区域、人体局部、相邻目标区域被误判为 person。
```

于是：

```text
FP 增加，Precision 下降。
```

这就是微调后的典型现象：

```text
模型从保守变得更激进。
```

## 9. 是否可能过拟合

目前存在一定过拟合风险。

原因：

```text
1. 训练集和验证集都来自 MOT17-02-SDP 同一个序列；
2. train 是连续帧 1-300，val 是连续帧 301-400；
3. 视频相邻帧高度相似；
4. 小数据集只有 300 张训练图；
5. 只训练 person 单类。
```

所以这次结果说明：

```text
模型适应了 MOT17-02-SDP 这个场景。
```

但还不能说明：

```text
模型对所有监控行人场景都泛化更好。
```

## 10. 下一步改进方向

### 10.1 调整 confidence threshold

微调后 FP 明显增加。可以尝试提高 `conf`：

```text
conf = 0.35
conf = 0.50
```

目标：

```text
保留较高 Recall，同时减少 FP，提高 Precision。
```

### 10.2 改进数据划分

当前划分是连续帧：

```text
train: 1-300
val: 301-400
```

后续可以尝试：

```text
train: 1-400
val: 401-600
```

或者使用多个 MOT17 序列：

```text
train: MOT17-02
val: MOT17-04
```

这样更能测试泛化。

### 10.3 加入更多训练数据

当前训练集较小：

```text
300 张图
8668 个框
```

可以扩大到：

```text
train 500+
val 100+
```

或加入其他 MOT17 序列。

## 11. Day17 结论

本次微调前后对比说明：

> 小规模 MOT17 微调显著提升了 YOLO11s 在 MOT17-02-SDP 上的 Recall，将 Recall 从 0.3644 提高到 0.6930，FN 大幅减少。但微调后模型也产生更多 FP，Precision 从 0.7681 下降到 0.6646。

当前结论：

```text
微调有效，但需要继续调 conf 和改进数据划分，控制 FP 并验证泛化能力。
```
