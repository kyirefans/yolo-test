## 一、理论和代码的关系要反过来理解

很多研究生容易犯一个错误：

> 先把理论全学完，再开始写代码。

这基本不可行。行为识别、目标检测、强化学习、VLA，每个方向都很深，等你“学完”再动手，可能半年过去了还没产出。

你应该采用：

> **先跑通，再理解，再改进。**

比如学目标检测，不要一开始纠结所有公式。正确顺序是：

```text
1. 跑通 YOLO 推理
2. 换自己的图片/视频测试
3. 看误检和漏检
4. 理解 mAP、NMS、Anchor / Anchor-free
5. 尝试微调数据集
6. 再去看 YOLO、DETR、Grounding DINO 的论文差异
```

学行为识别也是：

```text
1. 先跑通一个 SlowFast / VideoMAE demo
2. 输入自己的视频
3. 看模型容易把哪些行为混淆
4. 再理解时间建模、时空注意力、视频预训练
5. 最后考虑如何接入大模型做语义增强
```

学强化学习也一样：

```text
1. 先用 Gymnasium 跑 DQN / PPO
2. 看 reward、state、action 怎么设计
3. 再补 MDP、Bellman 方程、策略梯度
4. 最后再看机器人 RL、离线 RL、分层 RL
```

## 七、如何避免“理论学不深，代码也写不强”

你要给自己设几个硬标准。

### 理论学习的标准

看完一篇论文后，至少能回答：

```text
1. 它解决了什么问题？
2. 它和之前方法比，核心变化是什么？
3. 输入输出是什么？
4. 实验指标是什么？
5. 它有什么局限？
6. 我能不能把它用到自己的项目里？
```

不要求你一开始完全推公式，但不能只停留在“这篇论文很牛”。

------

### 代码学习的标准

跑通一个模型后，至少要做到：

```text
1. 知道入口文件在哪里
2. 知道数据怎么读入
3. 知道模型在哪里定义
4. 知道 loss / metric 在哪里计算
5. 知道怎么换数据集
6. 知道怎么保存和加载模型
7. 能解释一次推理流程
```

只会 `python demo.py` 不算真正掌握。

------

### 项目学习的标准

一个项目至少要有：

```text
README
环境配置
运行命令
模型说明
实验结果
失败案例分析
可视化截图或视频
后续改进方向
```

这样才可以用于导师汇报和实习简历。

## 八、LeetCode 不要被学术挤掉

你现在有学术任务，但实习面试还是要手撕代码。所以 LeetCode 要像健身一样每天做一点。

建议你保持：

```text
每天 1 题
每周 1 次复盘
每两周 1 次模拟面试
```

刷题顺序：

```text
数组 / 哈希 / 双指针
链表 / 栈 / 队列
树 / DFS / BFS
二分 / 堆
回溯 / 动态规划
图论 / 并查集
单调栈 / TopK
```

不要每天随机刷。你要按专题刷，形成模板。

## 九、AI Coding 怎么融入，而不是取代学习

Claude Code 和 Codex 可以帮你提效，但不能替你学习。你可以这样用：

### 可以让 AI 做的

```text
搭项目结构
解释代码库
生成测试
写接口文档
找 bug 可能位置
重构重复代码
生成 README 初稿
```

### 不应该完全交给 AI 的

```text
核心算法理解
模型结构判断
实验结论分析
面试题手撕
论文创新点判断
项目技术路线选择
```

你每次用 AI coding，都要问自己：

> **如果面试官让我解释这段代码，我能不能讲清楚？**

讲不清楚，就说明还没真正掌握。

## 十、最适合你的平衡原则

我建议你记住这句话：

> **代码保证你能找到实习，理论决定你读研的上限，项目把两者连接起来。**

所以你的优先级应该是：

```text
第一优先级：能写代码、能面试、能实习
第二优先级：有一个持续推进的研究型项目
第三优先级：围绕项目补理论，而不是漫无目的看论文
第四优先级：强化学习/VLA 作为中长期深入方向
```

你现在不需要每天读 5 篇论文，也不需要每天刷 10 道题。
更合理的是：

```text
每天 1 道题
每天推进一点项目
每周精读 1 篇论文
每周整理 1 次笔记
每月完成 1 个阶段性 demo
```

长期坚持 3—6 个月，你会明显超过那些“只看论文不写代码”或者“只刷题没有方向”的同学。
